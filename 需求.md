# TRPG战斗面板

## 主要模块
1. 新建、编辑敌人卡片模块，返回json格式
   - 以@敌人数据块.js中的数据格式，给用户显示一个手动输入敌人数据的窗口
   - 在其他模块中，编辑某敌人数据时，同样调用这个模块

2. 敌人库模块，以json存储敌人的信息，供用户选择
   - 默认加载 @敌人数据块.js 中的敌人数据
   - 在战斗面板中，左滑
   - 用户可以新建敌人，同样使用1.中的卡片模块，返回的json加入敌人库中
   - 用户可以上传json文件加入敌人库，也可以下载当前敌人库中的所有敌人为json文件
   - 以['位阶','种类','来源']对敌人库进行排序、筛选、多选

3. 战斗面板，从敌人库中选出敌人，把对应的json敌人数据以卡片的形式放在战斗面板上
   - 是默认入口界面
   - 卡片上有可点击标记的生命点、压力点等checkbox，可以用鼠标点击填充和擦除
   - 其他的json信息以文字形式写在卡片上
   - 敌人的特性只显示名字和类型，描述文本点击名字后显示。
   - 卡片具有一个备注框，可以直接在战斗面板上添加文字性注释
   - 面板上的每个卡片都是一个独立的json，可以右键点击，激活1.中的卡片模块进行修改
   - 支持导入导出当前战斗面板，包括上面的所有敌人卡片

## 技术要求
1. 使用html + css + js，部署在github pages等静态托管服务上，没有数据库和后端
2. 将敌人库、战斗面板储存在用户浏览器数据里，刷新、关闭浏览器不会丢失之前的数据
3. 尽量模块化，不同独立的模块分布在不同的文件里
4. 尽量轻量化，所有json数据都是文本字符串，不牵扯复杂的框架和应用
